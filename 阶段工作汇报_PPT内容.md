```java
@Around("@annotation(authCheck)")
public Object doInterceptor(ProceedingJoinPoint joinPoint,
        AuthCheck authCheck) {
    // 获取当前用户
    User loginUser = userService.getLoginUser(request);
    // 角色等级比较
    if (roleRank(userRoleEnum) < roleRank(mustRoleEnum)) {
        throw new BusinessException(ErrorCode.NO_AUTH_ERROR);
    }
    return joinPoint.proceed();
}
```

## 第 10 页：技术亮点 - 数据脱敏

### 🎯 数据脱敏处理

**脱敏规则：**

- **电话**：保留前 3 位和后 4 位，中间用\*\*\*\*代替
- **邮箱**：保留@前 1 位和@后部分
- **身份证号**：保留前 6 位和后 4 位

**动态脱敏：**

```java
// 如果当前登录员工是部门主管，则不对同事信息进行脱敏
if (!isCurrentEmployeeLeader) {
    // 执行脱敏处理
    vo.setPhone(phone.substring(0, 3) + "****"
        + phone.substring(phone.length() - 4));
}
```

**技术难点解决：**

- ✅ 根据角色和关系动态决定是否脱敏
- ✅ 脱敏规则设计合理，保留识别信息

## 第 11 页：技术亮点 - 批量操作优化

### 🎯 批量操作优化

**优化策略：**

- 避免 N+1 查询问题
- 使用批量更新方法
- 事务保障数据一致性

**核心实现：**

```java
// 先批量查询，避免重复查询
List<Employee> employeesToUpdate = new ArrayList<>();
for (Long employeeId : employeeIds) {
    employeesToUpdate.add(this.getById(employeeId));
}

// 使用MyBatis-Flex批量更新
UpdateChain.of(Employee.class)
    .set(Employee::getDepartmentId, departmentId)
    .where(Employee::getId).in(employeeIdList)
    .update();
```

**性能提升：**

- ✅ 减少数据库查询次数
- ✅ 批量操作事务保障

## 第 12 页：技术亮点 - 动态查询构建

### 🎯 权限动态查询构建

**技术亮点：**

- 根据用户角色动态构建查询条件
- 确保不同角色只能查看权限范围内的数据

**实现逻辑：**

```java
switch (roleEnum) {
    case ADMIN:
        // 管理员可以查看所有或指定公司的员工
        if (requestCompanyId != null) {
            qw.eq("company_id", requestCompanyId);
        }
        break;
    case COMPANY_ADMIN:
    case HR:
        // HR和公司管理员只能查看自己公司的员工
        qw.eq("company_id", loginUser.getCompanyId());
        break;
}
```

**优势：**

- ✅ 权限与业务逻辑解耦
- ✅ 数据库层面过滤，性能优化
