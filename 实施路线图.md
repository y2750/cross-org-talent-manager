# 跨组织人才管理系统 - 实施路线图

## 📋 快速概览

本文档提供了系统功能实施的优先级和步骤指南，帮助开发团队有序推进项目。

---

## 🎯 第一阶段：核心评价系统（2-3 周）

### 目标

实现基础的评价功能，包括标签化评价和维度评分。

### 任务清单

#### 1. 数据库准备（1 天）

- [ ] 执行 `database_improvements.sql` 脚本
- [ ] 修复 `evaluation` 表结构（添加 `employee_id` 字段）
- [ ] 创建评价标签表和维度表
- [ ] 初始化基础数据（维度、标签）

#### 2. 评价实体与 Mapper（2 天）

- [ ] 创建 `Evaluation` 实体类
- [ ] 创建 `EvaluationTag` 实体类
- [ ] 创建 `EvaluationDimension` 实体类
- [ ] 创建 `EvaluationDimensionScore` 实体类
- [ ] 创建对应的 Mapper 接口

#### 3. 评价服务层（3 天）

- [ ] 创建 `EvaluationService` 接口和实现
- [ ] 实现评价创建（包含标签和维度评分）
- [ ] 实现评价查询（按员工、按公司、按时间）
- [ ] 实现评价更新和删除
- [ ] 实现雷达图数据生成方法

#### 4. 评价控制器（2 天）

- [ ] 创建 `EvaluationController`
- [ ] 实现评价创建接口
- [ ] 实现评价查询接口
- [ ] 实现评价更新接口
- [ ] 实现雷达图数据接口

#### 5. 标签和维度管理（2 天）

- [ ] 创建 `EvaluationTagService` 和 `EvaluationDimensionService`
- [ ] 实现标签/维度的增删改查（仅管理员）
- [ ] 实现标签/维度列表查询接口

#### 6. 前端开发（5 天）

- [ ] 评价创建表单（标签选择、维度评分）
- [ ] 评价列表展示
- [ ] 评价详情查看
- [ ] 雷达图组件
- [ ] 标签和维度管理界面（管理员）

### 验收标准

- ✅ 可以创建包含标签和维度评分的评价
- ✅ 可以查看员工的评价列表和详情
- ✅ 可以查看员工的雷达图
- ✅ 管理员可以管理标签和维度

## 💰 第三阶段：积分系统（1-2 周）

### 目标

实现企业积分计算、查询和权益管理。

### 任务清单

#### 1. 数据库准备（0.5 天）

- [ ] 在 `company` 表中添加 `total_points` 字段（可选）
- [ ] 优化 `company_points` 表索引

#### 2. 积分服务（3 天）

- [ ] 创建 `CompanyPointsService`
- [ ] 实现积分计算规则：
  - 建立完整档案：+10 分
  - 收到员工好评：+5 分
  - 提供客观评价：根据评价质量加分
- [ ] 实现积分查询和统计
- [ ] 实现积分变动记录查询

#### 3. 积分控制器（1 天）

- [ ] 创建 `CompanyPointsController`
- [ ] 实现积分查询接口
- [ ] 实现积分变动记录查询接口
- [ ] 实现积分统计接口

#### 4. 积分权益（2 天）

- [ ] 实现优先检索人才库逻辑
- [ ] 实现首页推荐位逻辑
- [ ] 实现高级分析功能解锁逻辑

#### 5. 前端开发（3 天）

- [ ] 积分查询界面
- [ ] 积分变动记录展示
- [ ] 积分权益说明
- [ ] 积分统计图表

### 验收标准

- ✅ 企业积分可以正确计算和更新
- ✅ 可以查询积分和变动记录
- ✅ 积分权益功能正常工作
